Developer Reference

Namespaces & Code

Required namespaces and a quick look at SDK usage patterns.

ExcelReportGenerator.cs 
// Required namespaces
using DocumentFormat.OpenXml;
using DocumentFormat.OpenXml.Drawing
       .Spreadsheet;
using DocumentFormat.OpenXml.Office
       .Excel;
using DocumentFormat.OpenXml.Packaging;
using DocumentFormat.OpenXml.Spreadsheet;

// Open (or create) a spreadsheet
using (var doc =
  SpreadsheetDocument.Open(
    "Report.xlsx",
    true))
{
  var wb = doc
    .WorkbookPart.Workbook;

  // Access sheets, rows, cells…
}
StylesheetBuilder.cs 
// Build a stylesheet via SDK
var ss = new Stylesheet();
ss.Fonts = new Fonts(
  new Font(
    new Bold(),
    new FontSize {
      Val = 12
    }
  )
);
ss.Fills = new Fills(...);
ss.Borders = new Borders(...);

// Attach to workbook part
wbPart.WorkbookStylesPart
  .Stylesheet = ss;
Namespace Purpose
DocumentFormat.OpenXml Root namespace. Core types for all Open XML document manipulation.
…Drawing.Spreadsheet Handles drawing objects and charts embedded within spreadsheet documents.
…Office.Excel Excel-specific schema elements and extensions beyond the base OOXML spec.
…Packaging Opens, creates, and manages Open XML package containers (the ZIP layer).
…Spreadsheet All spreadsheet primitives: Workbook, Worksheet, Row, Cell, Stylesheet.
AUTO / MAN Mode

The Open XML SDK ships with a Document Reflector (AUTO) mode — paste in any .xlsx and it generates the full C# code that would produce it. Use MAN mode for precision control over every XML element when templates require custom logic beyond what the reflector outputs.
